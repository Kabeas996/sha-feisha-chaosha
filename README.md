# 三诀

轻量级网页双人实时心理博弈游戏。当前同时提供 Socket.IO 在线房间和本地同屏双人模式，全部复用服务器权威规则核心进行结算。

## 项目结构

```text
frontend/                 # React + Vite 本地试玩前端
  assets/icons/           # 七个技能及四种状态的透明 PNG
  src/assets/             # 前端素材清单与状态映射
backend/
  src/game-rules/         # 无网络、无 UI 依赖的纯 TypeScript 规则模块
  tests/                  # 规则单元测试
```

游戏状态与胜负只能由后端规则模块结算。未来客户端仅提交行动，不参与可信判定。

## 启动游戏

需要 Node.js 24+。首次使用在项目根目录执行：

```powershell
npm.cmd install
npm.cmd run dev
```

然后打开终端中 `WEB` 一行显示的地址：

- 在线对战：一位玩家创建六位房间码，另一位玩家从另一浏览器或设备加入，双方准备后开始。
- 同屏双人：一个玩家依次为玩家一、玩家二秘密选择行动。

在线服务器默认监听 `3001` 端口，前端默认连接 `http://127.0.0.1:3001`。部署或使用其他设备访问时，在 `frontend/.env` 中设置 `VITE_BACKEND_URL`，格式参考 `frontend/.env.example`。

## 当前实现

- 初始生命 3，初始能量 0，能量上限 5
- 石头无论是否受击都获得 1 点能量，且不超过上限
- 杀 / 飞杀 / 超杀分别消耗 1 / 2 / 3 点能量
- 同级攻击抵消，高级攻击压制低级攻击
- 防 / 下防 / 上防只阻挡对应等级攻击
- 伤害与胜负在同一个确定性纯函数中计算
- 非法能量状态、能量不足行动、结束后继续结算均会被拒绝
- 首页、玩法说明、玩家状态、技能选择和能量禁用状态
- 同屏双人的秘密选招交接流程
- 回合揭晓、伤害反馈、胜负弹窗与重新开局
- 手机和桌面响应式布局
- Socket.IO 创建房间、加入房间和准备状态
- 服务器统一三秒截止时间，客户端只显示服务器 deadline
- 未操作回合、连续三回合判负和十五秒断线重连
- 在线再战与单人视角界面
